A modern runtime for Edge Delivery Services

EDS Vite combines Vite, Preact, and TypeScript while preserving the semantic HTML authoring model that makes Adobe Edge Delivery Services approachable.

Vite powered

Fast development, hot module replacement, code splitting, and optimized production builds.

Progressive enhancement

Authored HTML remains meaningful before blocks add richer behavior.

Dynamic blocks

Blocks are discovered with import.meta.glob and loaded only when needed.

Preact where it helps

Interactive state uses focused Preact components; small decorators stay close to browser APIs.

Typed runtime

TypeScript keeps block contracts, utilities, and shared application state predictable.

Responsive media

Utilities support optimized pictures, image preloading, and responsive video.

Getting started

Install dependencies and start the Vite development server:

npm install
npm run dev

Create and inspect an optimized production build with:

npm run build
npm run preview

Project structure

src/
  main.ts       Page bootstrap and EDS runtime
  spectrum.ts   Adobe Spectrum registration and theme
  blocks/       Independently loaded page blocks
  components/   Shared Preact UI
  services/     Application data services
  mocks/pages/  Local authored-page fixtures
  utils/        Reusable authoring and media helpers
  styles/       Global layout styles
  assets/       Bundled images and icons

Page lifecycle

  1. Load authored content. The homepage comes from index.html; each local route receives its own generated HTML document containing the fragment from src/mocks/pages.
  2. Decorate the page. The runtime identifies sections, metadata, buttons, icons, and block wrappers.
  3. Load eager content. The first meaningful section and its media are prioritized for rendering.
  4. Load blocks. Each block module progressively enhances its authored markup and updates data-block-status.
  5. Load lazy content. Remaining sections, the header, and the footer complete after the critical page content.

Serving pages without an EDS backend

Each page source is ordinary EDS-shaped HTML. The page generator places it into the shared document shell, Vite builds it as a real HTML entry, and Vercel maps the clean route to that static document. No client-side HTML injection is used.

Remove the Vercel routes for real EDS projects.

The vercel.json rewrites are included only for Michael's free demo deployment. When an Adobe Edge Delivery Services backend is connected, remove that file and the demo static-route mapping so EDS owns page routing and returns the authored HTML.

Core philosophy

Authors continue to create semantic HTML. Vanilla TypeScript handles small transformations, Preact handles structured interactive interfaces, and shared behavior lives in utilities rather than being duplicated across blocks.
